# 项目名称：PixelPrompt (复古像素风交互版) 完整需求文档

## 1. 应用概述

* **应用名称**：PixelPrompt
* **核心描述**：一款极致致敬 1984 Macintosh System 1 美学的个人提示词管理工具。采用黑白像素风格，支持提示词的增删改查、本地存储、以及灵感碎片的快速记录。
* **核心体验**：**"Crunchy" (清脆感)**。拒绝现代网页的平滑过渡，所有交互反馈必须是瞬时、硬朗、高对比度的。

## 2. 核心设计哲学 (Design Specs)

* **色调**：**Monochrome**。纯黑 (#000000) 与 纯白 (#FFFFFF) 的极致对比。
* **纹理**：**Dithering**。背景必须使用高密度的“点阵/噪点”纹理，模拟老式显像管质感。
* **形状**：**No Radius**。全站拒绝圆角，坚持直角矩形的硬朗风格（Border-radius: 0）。
* **边框**：统一使用 1px 或 2px 纯黑实线边框。

## 3. 功能模块详解

### A. 顶部导航栏 (Header)

* **布局**：Flex 左右两端对齐，底部带 1px 黑色分割线，高度适中。
* **左侧**：LOGO "PixelPrompt" (使用像素字体，加粗)。
* **右侧工具栏**：
  1. **[关于] (i)**：点击弹出模态框 (Modal)，背景半透明网格遮罩。展示项目介绍，作者名显示为 **"Designed by kanshao 2077"** 。
  2. **[新建] (+)**：点击后在列表顶部插入一张空白卡片（进入编辑态）。
  3. **[数据管理]**：包含 [导入 JSON] 和 [导出 JSON] 两个像素风小按钮。

### B. 提示词手风琴列表 (Main List)

* **布局**：单列居中，最大宽度 800px。
* **默认状态 (Collapsed)**：
  * 样式：1px 黑边框矩形条。
  * 内容：[标题] (Bold) + [简述] (灰色，单行截断)。
  * 交互：点击卡片任意区域 -> 瞬间展开。
* **展开状态 (Expanded)**：
  * **编辑区**：点击标题和描述文字即可直接编辑。
  * **正文区**：多行文本框 (Textarea)，高度自适应，支持大段文本。
  * **操作反馈**：
    * **[复制] 按钮 (核心)**：点击后，按钮背景瞬间反转 (黑变白)，文字变 "COPIED!"，1秒后复原。
    * **[删除] 按钮**：位于角落，点击后需二次确认。
  * **自动保存**：所有输入框失去焦点 (onBlur) 时自动保存数据至 LocalStorage。

### C. 灵感碎片 (Memo Pad)

* **位置**：页面底部（或作为侧边悬浮面板）。
* **形态**：一个带有 "MEMO" 标题栏的独立框体，模拟“便签纸”视觉，使用虚线边框以示区分。
* **逻辑**：一个巨大的纯文本输入框，实时监听输入，内容变化即刻写入 LocalStorage。

## 4. 视觉与交互细节 (关键)

### A. 鼠标光标动态 (Cursor Dynamics)

* **强制自定义**：全站鼠标样式必须改为 **Retro Pixel** 风格。
  * 默认状态：大的像素箭头。
  * 悬停 (Hover) 链接/按钮：变为像素手指。
  * 输入文本：变为像素工字标 (I-Beam)。
* **技术限制**：请优先使用 CSS SVG Data URI 绘制，不要依赖外部图片链接。

### B. 悬停反馈 (Hover Effects)

* **反色机制 (Invert Strategy)**：
  * 当鼠标悬停在任何 **按钮** 或 **可点击卡片** 上时，**禁止使用透明度变化**。
  * **必须使用“反色”**：背景由白变黑，文字由黑变白 (`bg-black text-white`)。这种高对比度的闪烁感是复古风的精髓。

### C. 点击反馈 (Click/Active Effects)

* **机械位移**：
  * 当按钮被“按下” (Active 状态) 时，元素整体向右下方位移 1px-2px。
  * 配合黑色阴影 (Box-shadow) 的消失，模拟物理按键被按压的视觉差。

### D. 输入框特效

* **光标**：输入框内的打字光标 (Caret) 设为粗块状 (Block)，并带有复古的闪烁频率。
* **聚焦**：输入框激活 (Focus) 时，边框变为粗虚线或加粗至 2px。

## 5. 数据与存储逻辑

* **存储方案**：纯前端 `localStorage`，无后端。
* **Key 定义**：

  * `pixel_prompt_data`：存储提示词数组。
  * `pixel_memo_data`：存储灵感碎片字符串。
* **初始化逻辑 (关键)**：

  * 当检测到 `localStorage` 为空时，**必须**写入以下初始数据，避免白屏：

  ```json
  [
    {
      "id": "init_01",
      "title": "👋 欢迎使用 PixelPrompt",
      "desc": "这里是你的灵感弹药库 (点击展开)",
      "content": "这是一个复古风格的提示词管理工具。\n\n1. 点击右上角 [+] 新建提示词。\n2. 点击标题展开卡片。\n3. 点击 [复制] 按钮一键拿走内容。\n4. 底部是你的灵感草稿纸，随手记，自动存。\n\nEnjoy your creation!"
    }
  ]
  ```

## 6. 技术实现约束 (Technical Constraints)

为了确保视觉还原度，请严格执行以下 CSS 和资源策略：

1. **字体资源**：

   * 请在 Header 中引入 Google Fonts 的 **"VT323"** 或 **"Press Start 2P"**。
   * CSS 全局应用：`font-family: 'VT323', monospace;`
2. **背景纹理 (CSS Dithering)**：

   * 请直接使用 CSS 代码实现背景，**不要使用图片**：

   ```css
   body {
     background-color: #f0f0f0;
     background-image: radial-gradient(#000000 1px, transparent 1px);
     background-size: 4px 4px;
   }
   ```
3. **图标资源**：

   * 使用 **Lucide React** 或内联 SVG。**严禁**引用外部 `.png/.jpg` 链接。
4. **开发框架**：

   * 使用 **React** + **Tailwind CSS**。
   * 代码结构需整洁，确保单次复制即可运行。
